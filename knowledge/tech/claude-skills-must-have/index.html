⚙️ Skill的工作原理

"渐进式披露"三层架构：

  1. 第一层（元数据）：YAML格式的name+description，Claude启动时仅加载这部分到系统提示（约100 tokens/skill），不占上下文
  2. 第二层（核心指令）：Claude判断任务匹配后，按需加载完整的SKILL.md内容（操作步骤、注意事项、示例）
  3. 第三层（脚本+资源）：需要精确逻辑的任务，Skill目录内可包含 scripts/ 目录和 references/ 目录

💡 一个形象的比喻：MCP像"厨房+厨具"（连接能力），Skill是"菜谱"（做事方法）。光有厨房只能保证你能做饭，有菜谱才能保证你不炸厨房，而且每次做出来的味儿都一样稳定。

🎯 安装方式

方式命令

Claude Code /plugin/plugin marketplace add anthropics/skills
手动放置SKILL.md → ~/.claude/skills/<技能名>/ 或 .claude/skills/<技能名>/

💡 最佳实践

  • 别贪多：Skills是自动激活的，装多了Claude在不同skills之间做判断时容易冲突。官方推荐5-8个高频Skills足够
  • 信号要明确：请求里带上明确的信号词。说"提取这个PDF里的表格"比"帮我看一下这个文档"激活更准
  • 自己造最有价值：把你过去一周重复做了3次以上的工作拆出来，封装成自己的Skill——这是"会用AI"和"能带团队用AI"的分界线
  • skill-creator + find-skills 是入口：有了这两个元skill，后面对Skill的生态探索效率能翻倍
